Hero Codenjoy — how to play?

The game server is available for familiarization reasons http://codenjoy.com/codenjoy-contest

This is the open source game. To realize your game, correct errors in the current version and make the other corrections, you should fork the project at first. There is the description in the Readme.md file in the repository root. It is specified in the description what to do next.

If any questions, please write in skype:alexander.baglay or Email apofig@gmail.com

What is the gist of the game?

You should write your bot for the hero who will beat the other bots by the score it gets. All bots play on the same field of play. The hero can move by idle cells to all four directions.

The hero can plant a potion. The potion will explode in 5 ticks (seconds). The blast wave can affect inhabitants of the field. All affected by the blast wave disappear. You can decline by both your and someone else's potion.

On her/his way, the hero can meet a ghost – red air-ballon that destroys all heroes on its way.

Each destroyed object on the field (hero, ghost) or opened boxes is restored in an instant in the other place. If the hero is damaged, the penalty points (-50*) are allocated to him.

The hero whose potion destroyed something on the map receives bonus points as follows: +10* – for the opened boxes, +100* – for the ghost, +1000* – for the enemy hero.

*Please ask Sensei about scores on this game.

All points are summed up. The player with the largest number of points is considered to be a winner (prior to the due date).

So, the player registers on the server by indicating her/his Email.

Then you should connect from code to the server via Web Sockets. This is the Maven project and it will suit for playing in JVM languages. Inside this archive you can find also another languages. To know how to launch it, refer to the root of the README.txt.

For the other languages, you should write a native client (and then to share it with us by using Email: apofig@gmail.com)

The game is turn-based, every second the server sends to your client (bot) the state of the updated field at the moment and expects a response of the team to the hero. Within the next second, the player should give a command to the hero. If the player lost his/her chance, the hero remains in place.

The commands (UP, DOWN, LEFT, RIGHT) move the hero to the given direction in one cell; ACT – leave a potion on hero’s position. You can combine movement commands with the ACT command by separating them by a comma. The order (LEFT, ACT) or (ACT, LEFT) matters: either we move to the left and plant a potion there or we plant a potion and run away to the left. If a player uses the ACT command only, the potion will be planted under the hero without her/his move on the field.

The command АСТ(1). Used only with perk POISON_THROWER. Allows the hero to throw poisonous liquid. Works with the command to change direction. Order (LEFT, ACT(1)) or (ACT(1), LEFT) - does not matter. Without direction, just command ACT(1) nothing will happen. Hero just will stay on place.

The first task is to write a client’s WebSocket which will connect to the server. Then you should “force” the hero to listen to the commands. This is the way the gamer will prepare herself/himself to the main game. The main goal is to play meaningfully and win.

Up to now, several clients for playing are implemented, for the following programming languages: Java, JavaScript, Objective-C, Python, C#). The clients in the other languages are in the process of creation (thanks to the active gamers!).

The client code does not give a considerable handicap to gamers because you should spend time to puzzle out the code. However, it is pleasant to note that the logic of communication with the server plus some high level API for working with the board are implemented already.
